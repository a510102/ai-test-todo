# /debug — 找出問題根因並修復

## 問題描述
$ARGUMENTS

## 執行步驟

### Step 1：收集資訊
根據問題描述，讀取相關檔案：
- 錯誤訊息提到的檔案與行號
- 相關的 route handler、store、元件
- 相關的 Prisma schema（如涉及資料庫）
- 設定檔（如涉及環境變數或設定問題）

### Step 2：根因分析
輸出分析報告：

#### 症狀
描述觀察到的異常行為

#### 可能原因
列出 2-3 個最可能的根因，依可能性排序

#### 確認根因
說明哪個原因最符合症狀，以及判斷依據

### Step 3：修復方案
提出解決方案：
- **方案描述**：要修改什麼、如何修改
- **影響評估**：這個修改是否有副作用？
- **替代方案**：如果有更好的做法，一併說明

### Step 4：執行修復
修改對應的檔案，遵守各層的規範：
- 前端修改遵守 `frontend/CLAUDE.md`
- 後端修改遵守 `backend/CLAUDE.md`
- Schema 修改遵守 DB Agent 規則（`.claude/agents/db.md`）

### Step 5：驗證修復
說明如何確認問題已解決：
- 提供可以測試的步驟或指令
- 說明預期的正確行為

### Step 6：完成回報
- 列出修改的檔案
- 說明根本原因是什麼
- 如有必要，建議後續防範措施
